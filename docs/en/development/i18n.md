# Internationalization

The app uses `@ngx-translate/core` with a static translation loader. Runtime language is managed by `I18nService` and exposed through `LanguageSwitchComponent`.

## Supported languages

Translation files live in `src/app/core/i18n/translations`:

- `es.ts`
- `en.ts`
- `fr.ts`
- `pt.ts`
- `de.ts`

The fallback language in `app.config.ts` is Spanish (`es`).

## Runtime behavior

`I18nService`:

- Stores the selected language in browser storage.
- Cycles through supported languages.
- Applies language changes to `TranslateService`.
- Avoids browser-only side effects on the server.

`LanguageSwitchComponent` renders an accessible listbox-like language menu and closes on Escape/outside click.

## Adding or changing copy

1. Add the key to every translation file.
2. Keep key names grouped by feature domain.
3. Use `TranslatePipe` in templates:

```html
{{ 'profile.save' | translate }}
```

4. Use `TranslateService` in TypeScript when generating dynamic labels.
5. Update tests if text is asserted.

## Key naming

Recommended namespaces:

| Domain | Example |
| --- | --- |
| Common UI | `common.cancel`, `common.loading` |
| Navigation | `nav.dashboard`, `nav.profile` |
| Auth | `auth.login`, `register.*` |
| Profile | `profile.aiTitle` |
| Tasks | `tasks.clear`, `tasks.stats` |
| Chat | `chat.newConversation` |
| Tutorships | `tutorias.reserve.submit` |
| IA data | `datosIa.dropout.labels.course` |

## Documentation language

Application translations and documentation are separate:

- App UI copy lives in TypeScript translation files.
- VitePress documentation lives in Markdown under `docs`.
- English frontend documentation lives under `docs/en`.

## QA checklist

- No visible hardcoded text in new UI.
- New keys exist in all supported languages.
- Long translated strings wrap correctly on mobile.
- Icon-only controls have translated labels.
- Error messages explain how to recover.
