# Internacionalizacion

## Archivos

Las traducciones viven en:

```text
src/app/core/i18n/translations/
  es.ts
  en.ts
  fr.ts
  pt.ts
  de.ts
```

## Servicio

`I18nService` inicializa idioma y preferencia. `StaticTranslateLoader` carga objetos estaticos.

## Reglas

- Agregar la misma clave en todos los idiomas.
- Mantener la misma estructura de objetos.
- No hardcodear texto visible.
- Usar `TranslatePipe` en templates.
- Usar interpolacion para valores dinamicos:

```html
{{ 'tutorias.seatsAvailable' | translate: { count: slot.availableSeats, total: slot.availability.capacity } }}
```

## Validacion manual de claves

El proyecto no tiene todavia un script formal para comparar idiomas, pero se puede ejecutar una validacion con Node tomando `en.ts` como base y comparando `missing` y `stale` en los demas idiomas.

## Nombres de dominio

Usar claves por dominio:

- `auth.*`
- `nav.*`
- `profile.*`
- `tutorias.*`
- `tutor.*`
- `admin.*`
- `chat.*`
- `common.*`

Evitar mezclar textos de un modulo en otro si no son compartidos.
